Introduction

Being a field that is compatible to data mining methods and functional to real life, sports betting is an highly popular field in statistical learning practices. To this day, there are many machine learning, data mining research studies on betting aiming to forecast the match outcomes with accuracy and precision.

In this project we aimed to obtain predicted probabilities for outcomes of English Premier League matches. In this manner, we modeled this problem as a multi-class classification problem and utilized the subjects and methods we were taught during the lecture. The obtained probabilities we set out to define represent the predicted outcomes of matches under three possible cases: Home win, Tie or Away win.

Data we were given contained various features and statistics of matches and teams. In addition to these information, we were also provided with bookmakers odds for each match in the state space. Ultimately, we were tasked to best the bookmakers odds by providing better results in terms of match outcomes.

Performance metric defined for this project is called RPS (Ranked Probability Score) which yields an accuracy metric for predicted probabilities and their respective observed outcome by taking the ordinal nature of outcomes (Home-Tie-Away) into account. Calculation can be seen in the equation below where r is the number of outcomes, pj is the forecasted probability of outcome j and ej is the actual probability of outcome j.

Related Literature

We have conducted a moderate scale literature search on football betting using statistical models. We have found out that there are several classification methods most researchers use in common such as random forest and gradient boosting methods. What these studies differ from each other is their approach to explanatory feature extraction and selection.

ELO ratings are features representing the normalized strength of teams based on the results of their previous matches. Ratings for each team are set to a fixed value at the beginning of the training horizon and updated sequentially with the outcome of each match. Hence, by definition, ELO rating is a cumulative result record of teams that gains meaning as time progresses. Although Hvattum argue that ELO Ratings usage in scientific domain of machine learning is limited so far, there still are numerous non-academic studies that utilizes ELO practically. We decided to add several features on ELO in our feature matrix.

Buursma implemented the use of a certain number of historical features (latest n matches) for both teams of a match to predict the outcome probabilities. This method acts as a moving averaged feature generator that takes only the latest information into consideration. We found this approach quite useful as it is very logical to get a grip on and account for the most recent condition of a team. An overall highly successful team might consistently scoring bad in recent games and is expected to do bad for the next.

Although we are provided odd information for some games, we didnt use odd data as features since it is usually missing for some of the games we need to predict. Instead, we extracted some features related to last n games using the data from matches and stats files. Extracted features can be seen in the table below.

Lasso Regression

Lasso Regression is a powerful tool especially when it is used with multiple features. Since all our features extracted from last n games its is likely that they are somewhat correlated. Therefore, by penalizing the coefficients we aimed to come up with realistic predictions eliminating the risk of multicollinearity. To determine the parameter lambda we used 10 fold cross-validation with 2 replications. Trying out different combinations of features, we concluded to not use ELO ratings, ELO differences and ball possession as our features. All other features are used to build a lasso regression model with multinomial class prediction. In our case Lasso was the best model among others in terms of the minimum RPS for the test data.

Stochastic Gradient Boosting

Being an advanced tree-based approach, Stochastic Gradient Boosting is quite a suitable method for this task. Parameters to be tuned are number of trees, shrinkage factor and depth. Because of computational complexity, cross validation (10 folds, 2 replications) and parameter tuning took quite some time but we managed to get satisfactory results using this method in RPS performance.

Decision Trees

Decision trees are widely utilized in the area of data mining. Models created are usually high in descriptive power but it is necessary to tune its parameters to obtain effective results. Therefore we used 10 fold cross validation with 2 replications to determine the values of minimum bucket size and complexity parameters. Different parameter sequences are tried and the best parameter set in terms of RPS is chosen as the final model. Since cross validation is used for parameter tuning, model is tested on a test data apart from the training.

Random Forest

Another approach we employed was random forest. As in Homework 4, for the number of trees (J) we used 500 and for the minimal number of observations per tree leaf we used 5. For this model we tried to determine the optimum number of feature subset size per each iteration (m). Like the other models 10 fold cv with 2 replications is applied in order to get minimum RPS.

Lasso Combined With Random Forest

Since using different predictors would decrease the mean RPS we tried some convex combinations of probabilities generated by Lasso model and random forest model as predictors. It turns out that, for some proportion this combination provided best RPS values. Optimum proportion is determined by trying out different combinations for the coefficient p and (1-p)

Conclusions and Future Work

In conclusion, this project gave us the opportunity to learn and apply several data mining methods. We have tried five methods namely Lasso Regression, Stochastic Gradient Boosting, Decision Trees, Random Forest and some combination of those models. For each model we tuned parameters using 10 fold cross validation which turns out to be computationally expensive especially for Stochastic Gradient Boosting and Random Forest models. This can be improved by using other functions for SGB such as xgboost which provides faster results providing fathoming or parallel run options. During our project we usually kept parameter space narrow in order to get results quicker. thus number of tried parameters can be enlarged to get better insights as a future work.

As for combining models we only tried additive combinations of Lasso and Random Forest Models. Expanding this selection with SGB or applying some other ensemble learning methods such as bagging might be other options to obtain better RPS values for future work.

Although ELO ratings doesnt provide additional information for the penalized regression model it improved other models. Except from Ball Possession other extracted features seem important at least for some of the models. Same methods might be used to work with different features to generalize these results.

#Function intents to extract related features of a specific team using its last n match and stat data
extract_last_ngames <- function(matches,N=10){
  
  
  #Create dataframe to store extracted features
  L_n_matches<-cbind(matches,L_n_Homegoals= as.numeric(NA),L_n_Awaygoals=as.numeric(NA),L_n_Homeconceded= as.numeric(NA),
                     L_n_Awayconceded=as.numeric(NA),L_n_Homewins= as.numeric(NA),L_n_Awaywins=as.numeric(NA),
                     L_n_HomeELOchange= as.numeric(NA),L_n_AwayELOchange=as.numeric(NA),L_n_HomeGoalAttempts= as.numeric(NA),
                     L_n_AwayGoalAttempts= as.numeric(NA),L_n_HomeGoalkeeperSaves= as.numeric(NA), 
                     L_n_AwayGoalkeepersaves= as.numeric(NA),L_n_HomeBallPossession= as.numeric(NA),
                     L_n_AwayBallPossession= as.numeric(NA))
  
  # Home Team
  for (i in 1:length(L_n_matches$matchId)) {
      
      # Get home teams name and corresponding date
      h<- L_n_matches$Home[i]
      matchdate<- L_n_matches$Match_Date[i] 
      
      # Create last n games data
      x<-matches[(Home==h|Away==h)& Match_Date<matchdate ]
      x<-x[!is.na(Match_Result)]
      x<-x[order(Match_Date)]  
      
      # If team doesn't have last n data for the corresponding date skip the procedure (leave NA)
      if (length(x$matchId) < N) {next}
      
      # Obtain last n game wins,goals scored and conceded
      y<-tail(x,N)
      L_n_matches[i,]$L_n_Homegoals<-sum(as.numeric(y$Home==h)*y$Home_Score + as.numeric(y$Away==h)*y$Away_Score)
      L_n_matches[i,]$L_n_Homeconceded<-sum(as.numeric(y$Home==h)*y$Away_Score + as.numeric(y$Away==h)*y$Home_Score)
      L_n_matches[i,]$L_n_Homewins<-sum(as.numeric(y$Home==h)*y$Result_Home + as.numeric(y$Away==h)*y$Result_Away)
      
      # Obtain ELO difference between the first and nth game
      ELO1<-as.numeric(y[1,]$Home==h)*y[1,]$Home_Elo + as.numeric(y[1,]$Away==h)*y[1,]$Away_Elo
      ELON<-as.numeric(y[N,]$Home==h)*y[N,]$Home_Elo + as.numeric(y[N,]$Away==h)*y[N,]$Away_Elo
      L_n_matches[i,]$L_n_HomeELOchange<-ELON-ELO1
      
      # For stat data merge dataframes to obtain last n games' statitstics
      z<-merge(x,statdata[,list(matchId,home_GoalAttempts,away_GoalAttempts,home_GoalkeeperSaves,away_GoalkeeperSaves,home_BallPossession,away_BallPossession)], by = "matchId")
      z=z[complete.cases(z)==TRUE]
      
      # If team doesn't have last n data for the corresponding date skip the procedure (leave NA)
      if (length(z$matchId) < N) {next}
      z<-tail(z,N)
      
      # Obtain last n game goal attempts, goalkeeper saves and ball possession
      L_n_matches[i,]$L_n_HomeGoalAttempts<-sum(as.numeric(z$Home==h)*z$home_GoalAttempts+as.numeric(z$Away==h)*z$away_GoalAttempts)
      L_n_matches[i,]$L_n_HomeGoalkeeperSaves<-sum(as.numeric(z$Home==h)*z$home_GoalkeeperSaves+as.numeric(z$Away==h)*z$away_GoalkeeperSaves)
      L_n_matches[i,]$L_n_HomeBallPossession<-sum(as.numeric(z$Home==h)*z$home_BallPossession+as.numeric(z$Away==h)*z$away_BallPossession)
      
  }
  
  # Away Team
  for (i in 1:length(L_n_matches$matchId)) {
    
    # Get away teams name and corresponding date
    a<- L_n_matches$Away[i]
    matchdate<- L_n_matches$Match_Date[i]
    
    # Create last n games data
    x<-matches[(Home==a|Away==a)& Match_Date<=matchdate ]
      x<-x[!is.na(Match_Result)]
      x<-x[order(Match_Date)]  
      
      # If team doesn't have last n data for the corresponding date skip the procedure (leave NA)
      if (length(x$matchId) < N){next}
      
      # Obtain last n game wins,goals scored and conceded
      y<-tail(x,N)
      L_n_matches[i,]$L_n_Awaygoals<-sum(as.numeric(y$Home==a)*y$Home_Score + as.numeric(y$Away==a)*y$Away_Score)
      L_n_matches[i,]$L_n_Awayconceded<-sum(as.numeric(y$Home==a)*y$Away_Score + as.numeric(y$Away==a)*y$Home_Score)
      L_n_matches[i,]$L_n_Awaywins<-sum(as.numeric(y$Home==a)*y$Result_Home + as.numeric(y$Away==a)*y$Result_Away)
      
      # Obtain ELO difference between the first and nth game
      ELO1<-as.numeric(y[1,]$Home==a)*y[1,]$Home_Elo + as.numeric(y[1,]$Away==a)*y[1,]$Away_Elo
      ELON<-as.numeric(y[N,]$Home==a)*y[N,]$Home_Elo + as.numeric(y[N,]$Away==a)*y[N,]$Away_Elo
      L_n_matches[i,]$L_n_AwayELOchange<-ELON-ELO1
      
      
      # For stat data merge dataframes to obtain last n games' statitstics
      z<-merge(x,statdata[,list(matchId,home_GoalAttempts,away_GoalAttempts,home_GoalkeeperSaves,away_GoalkeeperSaves,home_BallPossession,away_BallPossession)], by = "matchId")
      z=z[complete.cases(z)==TRUE]
      
      # If team doesn't have last n data for the corresponding date skip the procedure (leave NA)
      if (length(z$matchId) < N) {next}
      z<-tail(z,N)
      
      # Obtain last n game goal attempts, goalkeeper saves and ball possession
      L_n_matches[i,]$L_n_AwayGoalAttempts<-sum(as.numeric(z$Home==a)*z$home_GoalAttempts+as.numeric(z$Away==a)*z$away_GoalAttempts)
      L_n_matches[i,]$L_n_AwayGoalkeepersaves<-sum(as.numeric(z$Home==a)*z$home_GoalkeeperSaves+as.numeric(z$Away==a)*z$away_GoalkeeperSaves)
      L_n_matches[i,]$L_n_AwayBallPossession<-sum(as.numeric(z$Home==a)*z$home_BallPossession+as.numeric(z$Away==a)*z$away_BallPossession)
